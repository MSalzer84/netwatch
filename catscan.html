CATScan Monitoring

Ein selbst entwickeltes Netzwerk-Monitoring-System — skalierbar für Heimnetzwerke wie für Firmeninfrastrukturen. Rechner, Server, Drucker, USVs, Access Points und Hypervisoren aus verschiedenen Netzen und Standorten werden in einem übersichtlichen Live-Dashboard zusammengefasst — ähnlich wie Zabbix, aber ohne Lizenzkosten und vollständig unter eigener Kontrolle.

1000+ Geräte Multi-Subnet Multi-Standort WebSocket Live In Betrieb Node.js Backend PowerShell · Python Agent
✅ Live in Betrieb: Mehrere Agenten senden alle 30–60 Sekunden Messdaten — CPU · RAM · Disk · Ping · Temperatur · Bandbreite · Paketverlust. Node.js-Server auf Port 3000, WebSocket auf 3001, SQLite-Datenbank. Proxmox- und Hyper-V-Hosts werden ohne Agent über die jeweilige API abgefragt.

Was CATScan ist

CATScan überwacht alle Geräte im Netzwerk — PCs, Server, Drucker, USVs, Switches und Access Points — und stellt sie in einem Live-Dashboard übersichtlich dar. Die Geräte lassen sich in einer frei benennbaren Baumstruktur nach Standort, Netzwerk und Gruppe organisieren.

Pro Gerät können beliebig viele Sensoren eingerichtet werden: CPU-Last, RAM, Festplattenauslastung, Ping-Latenz, Paketverlust, Bandbreite, Temperatur, Batteriestatus oder SSL-Zertifikat-Ablauf. Jeder Sensor hat eigene Warn- und Kritisch-Schwellwerte — bei Überschreitung erscheint sofort eine Push-Benachrichtigung.

Geräte ohne Agent — Drucker, USVs, Switches — werden per SNMP direkt vom Server abgefragt, kein Software-Install nötig. Proxmox- und Hyper-V-Hosts werden über ihre APIs eingebunden: alle VMs und Container erscheinen automatisch mit Status, CPU- und RAM-Auslastung.

Zusätzlich steht ein Kiosk-Modus zur Verfügung — eine Vollbild-Ansicht mit großen Statuskacheln, konfigurierbaren Anzeigebereichen und einem Alert-Ticker am unteren Rand.

Umgesetzte Features

Flexibler Geräte-Baum ✓ Fertig
Die Baumstruktur ist frei benennbar. Ebenen können hinzugefügt, umbenannt und neu angeordnet werden — per Doppelklick direkt im Browser, ohne Neustart.
Sensor-Konzept pro Gerät ✓ Fertig
Jedes Gerät hat einzelne Sensoren mit eigenem Status: CPU-Last, RAM-Auslastung, Disk, Ping-Latenz, Temperatur, Bandbreite, Paketverlust und SSL-Zertifikat-Ablauf. Schwellwerte und Warnzeiten sind frei einstellbar.
Windows Agent (PowerShell) ✓ Fertig
Läuft als PowerShell-Script auf jedem Windows-PC oder Server. Sammelt CPU, RAM, Disk, Ping, Temperatur, Batterie und Hyper-V VM-Status via WMI. Sendet alle 30 Sekunden an das Backend. Einrichtung per Doppelklick auf eine vorbereitete .bat-Datei.
Mac / Linux Agent (Python) ✓ Fertig
Python 3 Script ohne externe Abhängigkeiten. Läuft auf jedem Mac und Linux-System — Python ist dort vorinstalliert. Meldet sich beim Start automatisch am Backend an, kein manuelles Eintragen nötig.
SNMP-Poller für Drucker, USVs, APs ✓ Fertig
Geräte ohne Agent (Drucker, USVs, Access Points, Switches) werden per SNMP vom Server abgefragt. Bandbreite, Fehlerrate und gerätespezifische Werte — kein Agent nötig, Standard-SNMP-Community reicht.
Multi-Subnet & Multi-Standort ✓ Fertig
Agenten aus verschiedenen VLANs und Standorten schicken ihre Daten über das Netzwerk oder VPN ans Backend. Für Standorte ohne VPN gibt es einen lokalen Relay-Server der Daten puffert und weiterleitet.
Push-Benachrichtigungen ✓ Fertig
Bei kritischen Ereignissen (Gerät offline, CPU über Schwellwert, Sensor kritisch) kommt eine Push-Meldung direkt aufs Smartphone. Kostenlos über ntfy.sh, keine App-Registrierung nötig.
Kiosk-Modus für TV-Displays ✓ Fertig
Vollbild-Ansicht für Monitore in Server-Umgebungen. Frei wählbare Bereiche als große Statuskacheln mit CPU, RAM, Ping, Temperatur und VM-Übersicht. Alert-Ticker am unteren Rand, konfigurierbare Anzeige-Toggles.
Auto-Discovery ✓ Fertig
IP-Bereich scannen und alle antwortenden Geräte automatisch im Dashboard anzeigen — mit Hostname, MAC-Adresse und Hersteller-Erkennung über ARP. Agenten melden sich beim ersten Start selbst am Backend an.
Hypervisor & VM-Erkennung ✓ Fertig
Proxmox und Hyper-V werden direkt über ihre APIs abgefragt — kein Agent in den VMs nötig. VMs und Container werden mit Status, CPU- und RAM-Auslastung im Dashboard angezeigt und im Kiosk-Modus eingeblendet. Proxmox-Host-Metriken (CPU, RAM, Disk) werden automatisch als Gerätedaten übernommen.
Bandbreite & Paketverlust ✓ Fertig
Empfangs- und Senderate in kbps/Mbps werden per SNMP (ifInOctets/ifOutOctets) oder direkt vom Agent gemeldet. Paketverlust wird aus mehreren parallelen Pings ermittelt und farblich bewertet.
SSL-Zertifikat-Überwachung ✓ Fertig
Prüft TLS-Zertifikate auf Ablaufdatum — konfigurierbar mit Warn- und Kritisch-Schwellwert in Tagen. Meldet verbleibende Restlaufzeit als Sensor, gibt rechtzeitig eine Push-Benachrichtigung vor dem Ablauf.

Ausschnitt aus dem CATScan Dashboard (v1.0)

Eigenen Server einrichten

CATScan läuft auf jedem Rechner mit Node.js — Windows, Linux oder Raspberry Pi. Es braucht keine Datenbank-Installation, keine Cloud und keinen externen Dienst. Alles läuft lokal im eigenen Netzwerk.
1️⃣
Node.js installieren
Voraussetzung · einmalig
Node.js (ab Version 18) von nodejs.org herunterladen und installieren. Danach im Terminal prüfen:
node --version
2️⃣
Repository klonen
GitHub · git oder ZIP
Projekt von GitHub holen und Abhängigkeiten installieren:
git clone https://github.com/MSalzer84/catscan.git
cd catscan
npm install
Kein git? Oben rechts auf GitHub → Code → Download ZIP, entpacken, Terminal im Ordner öffnen, npm install ausführen.
3️⃣
Server starten
läuft auf Port 3000
node server.js
Das Dashboard ist danach erreichbar unter http://<DEINE-IP>:3000/netwatch-v3.html — von jedem Gerät im Netzwerk.
4️⃣
Dauerhaft laufen lassen
optional · empfohlen
Damit der Server nach einem Neustart automatisch startet:
Windows — geplante Aufgabe:
schtasks /create /tn "CATScan" /tr "node C:\catscan\server.js" /sc onstart /ru SYSTEM /f
Linux — systemd:
npm install -g pm2
pm2 start server.js --name catscan
pm2 save && pm2 startup

Agent installieren

Der Agent läuft im Hintergrund und sendet alle 30–60 Sekunden Messdaten an den CATScan-Server. Kein Konto nötig, keine Cloud — alles bleibt im eigenen Netzwerk. Auf Windows reicht ein Doppelklick, auf Linux und Mac ein einzelner Terminal-Befehl.
🪟
Windows
PowerShell · ab Windows 10
1
Installer herunterladen — die Server-Adresse ist bereits eingebaut
2
Doppelklick auf install-agent.bat (als Administrator)
3
Agent läuft sofort und erscheint im Dashboard
⬇ install-agent.bat herunterladen
Richtet den Agent als geplante Windows-Aufgabe ein — startet automatisch mit dem System. Kein manueller Neustart nötig.
Oder manuell per PowerShell:
iwr http://CATSCAN-SERVER:3000/agents/agent.ps1 -OutFile agent.ps1
powershell -File agent.ps1 -Server http://CATSCAN-SERVER:3000
🐧
Linux & Mac
Python 3 · vorinstalliert
1
Terminal öffnen
2
Einen Befehl einfügen und Enter drücken
3
Agent läuft und erscheint im Dashboard
One-Liner (curl):
curl -s http://CATSCAN-SERVER:3000/agents/linux-agent.py | python3 - --server http://CATSCAN-SERVER:3000
Für dauerhaften Betrieb als systemd-Dienst einrichten — Anleitung im Dashboard unter Einstellungen → Agent.
Dauerhaft (systemd):
curl -s http://CATSCAN-SERVER:3000/agents/linux-agent.py -o /opt/catscan-agent.py
python3 /opt/catscan-agent.py --server http://CATSCAN-SERVER:3000 &
🖧
Drucker, Switches, USVs — kein Agent nötig
SNMP · Proxmox API · Hyper-V WMI
Netzwerkgeräte ohne Windows oder Linux werden per SNMP direkt vom CATScan-Server abgefragt — SNMP-Community im Dashboard eintragen, fertig.

Proxmox-Hosts werden über den API-Token eingebunden: Im Dashboard das Gerät öffnen → Hypervisor-Konfiguration → Typ proxmox, URL und Token eintragen. VMs und Container erscheinen danach automatisch.

Technologie

CATScan ist bewusst einfach gehalten — kein Cloud-Zwang, keine teuren Lizenzen. Das Backend läuft auf einem normalen Windows Server oder Raspberry Pi. Das Dashboard ist eine einzige HTML-Datei die sich per WebSocket mit dem Backend verbindet.
